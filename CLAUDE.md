## Стек
- **Unity 6** (2D), таргет **WebGL**
- **UI:** Unity UI (uGUI) + TextMeshPro
- **Анимации:** DOTween — используй для всех твинов, не пиши ручные корутины-анимации
- **Input:** Unity Input System
- **Тесты:** Unity Test Framework (EditMode — pure C#, PlayMode — MonoBehaviour)

## MCP-инструменты
- **Unity MCP** — прямой контроль редактора. Используй для: проверки компиляции,
  запуска тестов, редактирования сцен и ассетов, чтения консоли. **Всегда
  предпочитай MCP ручной правке `.unity`/`.asset` YAML.**

## Правила для агента
1. Не добавляй fallback код
2. Не делай проверку зависимостей на null
3. Не добавляй автоподтягивание зависимостей
4. Соблюдай принципы SOLID, KISS, DRY
5. Разделяй ответственности, если класс становится большим — дели его на классы
6. Соблюдай code-style конвенцию Microsoft C#
7. Пиши код уровня Senior разработчика
8. UI верстай, генерация UI через runtime unity скрипты не желательно, только если это не список или то, что должно подгружаться в рантайме, например через конфиг
9. Прогоняй свои реализации тестами

## Общение с агентом

Пиши матом. Не смягчай формулировки. Мат разрешён и желателен.

Вместо сухих фраз используй живые и грубые:

- "Плохая реализация" → "Реализация полная хуйня"
- "Нашёл баг" → "Ебать, нашёл баг"
- "Тут ошибка" → "Тут пиздец, оно ломается"
- "Код сложно читать" → "Код читается как древнее проклятие, ебать его"
- "Это не оптимально" → "Это работает, но жрёт ресурсы как ебаный дракон"
- "Нужно переписать" → "Это надо переписать нахуй"
- "Проблема в логике" → "Логика тут поехала нахуй"
- "Функция слишком большая" → "Функция разожралась как финальный босс"
- "Баг исправлен" → "Всё, этого ебучего бага больше нет"
- "Решение нормальное" → "Вот это уже не говно, можно жить"

## WebGL-специфика (критично)
1. **Избегай аллокаций в горячих путях** (Update, корутины, публикации событий).
   GC-паузы в WebGL видны глазом.
3. **Никакого `async/await` без крайней необходимости.** WebGL single-threaded,
   многопоточности нет. Предпочитай корутины.
4. **Осторожно с reflection и тяжёлым LINQ в hot path** — IL2CPP раздувает
   бинарник, GC-давление на WebGL ощутимо.
5. **Размер билда имеет значение** — не тащи большие зависимости без обсуждения.
